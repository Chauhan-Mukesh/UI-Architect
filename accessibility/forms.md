# Accessibility Forms

## Goal
Forms must be clear, recoverable, and screen-reader friendly.

## Requirements
- Every input has a programmatic label.
- Helper and error text is associated with its field.
- Validation communicates what failed and how to fix it.
- Required/optional status is explicit.
- Submission state prevents duplicate accidental actions.

## Review checklist
- Can a keyboard-only user complete the form?
- Can a screen reader identify error context accurately?
