# Design Principles

## Objective
Define reusable UI/UX principles that guide all generated interfaces toward clarity, trust, and conversion.

## Core principles
1. **Purpose-first structure**: every section must support a business or user goal.
2. **Progressive disclosure**: show essential information first, then details.
3. **Consistency over novelty**: interactions should feel predictable across the page.
4. **Feedback and affordance**: interactive elements must communicate state and action.
5. **Error prevention first**: design forms and flows to reduce mistakes before validation.

## Practical checks
- Can users identify page purpose in under 5 seconds?
- Does each primary action appear once with clear priority?
- Are spacing, alignment, and component patterns consistent?
- Are empty/error/loading states designed intentionally?

## Anti-patterns
- Decorative sections without user value.
- Multiple competing primary CTAs in one viewport.
- Dense text blocks without hierarchy or grouping.
