# Information Architecture

## Objective
Define content and navigation structure that maps to user tasks and business outcomes.

## IA process
1. Identify primary user goals and top tasks.
2. Inventory required content and actions.
3. Group by mental model, not internal team structure.
4. Define navigation depth and cross-links.
5. Validate with top journeys and edge cases.

## Output artifacts
- Sitemap with priority levels.
- Page purpose statements.
- Navigation schema (global/local/contextual).
- Entry/exit points for key conversion flows.

## Quality checks
- Any critical task reachable in <= 3 interactions.
- Labels understandable without internal jargon.
