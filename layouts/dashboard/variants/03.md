# Dashboard Variant 03 — Multi-Panel Operational

## Structure
- Left rail for navigation (compact).
- Center: primary data (charts, tables, lists).
- Right: inspector or secondary data.
- Bottom (optional): logs, timeline, or activity.

## Use when
- Analytics, observability, trading, ops consoles.
- User needs multiple data streams visible simultaneously.

## Grid & spacing
- Panels resizable; remember user preferences.
- Dense typography and controls; base font size may shift down one step.

## Breakpoints
- Mobile: not supported for high-density ops surfaces; provide a limited view or explicit desktop-only banner if appropriate.
- Tablet: two-panel split; deprioritize the bottom panel.
- Desktop: full four-region layout.

## Accessibility
- Each panel is a landmark with a labeled region.
- Focus can move between panels via a keyboard shortcut (documented).
- Density does not sacrifice minimum touch/click sizes for interactive controls.

## Anti-patterns
- Panels that can be resized to zero without a restore.
- Charts as decorative fills without semantic labeling.
