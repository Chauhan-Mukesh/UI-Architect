# Dashboard Variant 01 — Sidebar + Workspace

## Structure
- Persistent left sidebar with grouped navigation (workspace switcher, primary nav, secondary nav, user menu at bottom).
- Main workspace: top toolbar with contextual actions, then the content grid or list.
- Optional right rail for inspector / details when an item is selected.

## Use when
- App has 4+ top-level areas.
- Users pivot between areas frequently.

## Grid & spacing
- Sidebar 240–280px on desktop; 60–72px collapsed rail alternative.
- Workspace uses a max width only when content is inherently narrow (settings). Otherwise fluid.

## Breakpoints
- Mobile ≤768: sidebar becomes a full-screen menu triggered by a button.
- Tablet: sidebar collapses to icon rail.
- Desktop ≥1200: full sidebar; inspector rail available.

## Accessibility
- Sidebar: `<nav aria-label="Primary">`, current item `aria-current="page"`.
- Skip-link to workspace.
- Keyboard: focus follows navigation on route change.

## Anti-patterns
- Sidebar with more than 7 primary items at one level.
- Icon-only rail without tooltips or labels.
