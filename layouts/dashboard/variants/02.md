# Dashboard Variant 02 — Top Nav + Tabs

## Structure
- Top nav with brand, primary areas (3–5 tabs), and user menu.
- Below top nav: subtabs or breadcrumbs for the active area.
- Main workspace fills the rest.

## Use when
- App has ≤5 top-level areas.
- Users spend most time within one area at a time.

## Grid & spacing
- Top nav ~56–64px tall.
- Subtabs align to the workspace content edge, not the viewport edge.

## Breakpoints
- Mobile: top nav collapses to a menu; subtabs may become a horizontal scroll.
- Desktop: both layers persistent.

## Accessibility
- Tabs use ARIA tabs pattern only if they behave as tabs (single view swap). Otherwise use links.
- Ensure keyboard order: nav → subtabs → main.

## Anti-patterns
- Cramming five levels of navigation into a top bar.
- Subtabs that shift when hovered — layout stability is a keyboard cost.
