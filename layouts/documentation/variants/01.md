# Documentation Variant 01 — Three-Column

## Structure
- Left: site nav (versioned if applicable).
- Center: article content.
- Right: in-page table of contents (H2s and H3s).
- Top: search + version switcher + light/dark toggle.

## Use when
- Long reference documentation.
- Readers jump within pages as much as between pages.

## Grid & spacing
- Left nav 240–280px; TOC 220–260px; content flexes.
- Content max-width around 720px for legibility.

## Breakpoints
- Mobile: nav becomes a drawer; TOC hidden or as a floating button.
- Tablet: hide TOC.
- Desktop: all three columns.

## Accessibility
- TOC is `<nav aria-label="On this page">`.
- Sticky headers do not obscure anchor targets on jump.

## Anti-patterns
- TOC that duplicates the entire page nav.
- Search that returns marketing pages before docs.
