# Documentation Variant 03 — Single-Column Tutorial

## Structure
- Full-width top header with title, estimated time, prerequisites.
- Single-column linear content: intro, step 1, step 2, …, check, next.
- Persistent progress indicator (optional).
- Prev/next at the bottom, plus link back to the tutorial index.

## Use when
- Onboarding paths and hands-on tutorials.
- Reader is expected to follow linearly, not jump.

## Grid & spacing
- Content max-width 65–75ch.
- Code blocks may extend wider than prose.

## Breakpoints
- Mobile and desktop use the same single-column pattern.

## Accessibility
- Estimated time and prerequisites announced in the heading region.
- Each step is a heading with a stable ID.
- Interactive check-your-work sections are keyboard-usable.

## Anti-patterns
- Hiding step count until the end.
- Auto-scrolling past code blocks.
