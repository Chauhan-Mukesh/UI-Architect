# Landing Page Layouts

## Purpose
Convert a specific audience toward a specific action. Every section earns its place by contributing to the decision.

## When to use which variant
- **01 Classic conversion** — proven high-intent flows (SaaS, professional services).
- **02 Story-driven** — narratives requiring context (nonprofits, complex offerings, launches).
- **03 Product-first** — where the product's surface is the strongest evidence.

## Universal rules
- Exactly one primary conversion goal. State it in the scope brief; every section serves it.
- Section sequence follows the visitor's decision journey, not a template rhythm.
- No section repeats the same message twice. If it does, cut one.
- Trust and objection-handling appear before the final CTA.

## Anti-patterns
- Generic hero → 3-card grid → CTA → footer.
- Feature tour where every "feature" is a phrase, not a benefit.
- Testimonials without attribution or role context.
- Pricing surprises hidden below the fold that were absent above.

## Accessibility
- Every CTA labeled with verb+object.
- Anchor navigation (if used) reflects real section hierarchy.
- Skip-link to main content.

## Responsive
- Mobile section order can differ from desktop when the decision path benefits.
- CTAs remain reachable on scroll (sticky or repeated).
