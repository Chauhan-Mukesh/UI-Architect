# Landing Variant 03 — Product-First

## Structure
1. Hero with product surface (screenshot, demo, or live embed) + short pitch
2. Guided feature tour: 3–5 modules each showing a real product view
3. Integrations or ecosystem strip
4. Pricing overview (or link to pricing)
5. Docs / evaluation entry point
6. Trust: security, uptime, compliance if relevant
7. Final CTA (start / talk to team / try demo)

## Use when
- Product is the argument. Developers, technical evaluators, or design tools.
- Screenshots or embedded product feel more convincing than any copy.

## Grid & spacing
- Product visuals get the largest components; copy sits in structured columns beside or below.
- Consistent framing (browser chrome, device frame) across features.

## Breakpoints
- Mobile: keep screenshots readable — crop to the key region rather than fitting the whole UI.
- Desktop: allow full-bleed screenshots with generous margins.

## Accessibility
- Every product screenshot has alt describing what the visible surface does.
- Interactive embeds handle focus trapping and keyboard exit.

## Anti-patterns
- Marketing UI mockups that don't match the real product.
- Feature grid without any visible product surfaces.
