# Landing Variant 01 — Classic Conversion

## Structure
1. Hero with value + primary CTA
2. Social proof strip
3. Problem framing / for-whom segmentation
4. How it works (3–5 steps or capability groups)
5. Deeper proof (case study, metric block, testimonial with attribution)
6. FAQ (objection handling)
7. Final CTA with restated value

## Use when
- Conversion goal is clear (trial, demo, signup) and audience is decided.
- The pitch can be told in a linear sequence.

## Grid & spacing
- 12-column grid on desktop; content max-width ~1200px.
- Uniform vertical rhythm between sections; use rhythm to signal progression.

## Breakpoints
- Mobile: single column; sticky bottom CTA optional if scroll is long.
- Tablet: single column with wider content area.
- Desktop: 12-column; features/steps may go two-up or three-up.

## Accessibility
- Section landmarks: `<section aria-labelledby>` with visible headings.
- FAQ uses `<details>/<summary>` unless interaction requires more.

## Anti-patterns
- Repeating identical CTA every section.
- FAQ items that are marketing copy pretending to be questions.
