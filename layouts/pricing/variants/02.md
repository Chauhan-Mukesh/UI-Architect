# Pricing Variant 02 — Feature Matrix

## Structure
- Compact tier summary at top (name, price, CTA).
- Detailed matrix below: rows are features grouped by category, columns are plans.
- Cells show yes/no, quantity, or short qualifier.

## Use when
- Many features; buyers compare across ≥3 plans.
- Enterprise sales-adjacent (procurement wants a spreadsheet feel).

## Grid & spacing
- Sticky header row on scroll.
- Row hover highlights the entire row.

## Breakpoints
- Mobile: convert matrix to per-tier accordion, keeping the same feature order.
- Desktop: full matrix.

## Accessibility
- Real `<table>` with `<thead>`, `<th scope="col">` for plans, `<th scope="row">` for features.
- Icons in cells always paired with text (`aria-hidden` on icon, visible label).

## Anti-patterns
- Icons alone conveying meaning ("check" vs "dash" only).
- Sticky headers that overlap and hide cells at small heights.
