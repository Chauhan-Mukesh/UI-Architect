# Hero Variant 02 — Split (Copy + Proof)

## Structure
- Two-column composition: left column carries headline/subhead/CTA; right column carries a product screenshot, dashboard preview, or single strong image.
- Both columns share vertical center on desktop; stack on mobile with copy first.

## Use when
- Product is visually explainable (dashboards, apps, tools).
- Screenshot or single frame carries meaningful proof.

## Grid & spacing
- Two-column grid `1fr 1fr` or `1.1fr 0.9fr` on desktop.
- Image column has slight overflow allowed off the right edge to imply scale.

## Breakpoints
- Mobile: copy column then image column. If fold pressure is severe, delay image to below the fold.
- Tablet: single column stacked with wider image.
- Desktop: split as above; image never dominates copy weight.

## Accessibility
- Image is `<img>` with meaningful `alt` describing the product surface shown.
- If the "image" is an interactive demo, it lives in a landmark region.

## Anti-patterns
- Screenshot that requires zoom to understand.
- Placeholder or lorem UI in the demo image.
- Image right-aligned so tightly that it fights the CTA on medium screens.
