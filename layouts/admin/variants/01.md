# Admin Variant 01 — CRUD List + Detail

## Structure
- Left rail: navigation for entity types.
- Main list: filter row (search, filters, sort), table of entities, pagination.
- Right panel or route: detail view for the selected entity with tabs (overview, settings, activity).
- Toolbar: create, bulk actions, export.

## Use when
- Managing users, projects, resources, or any collection with individual records.

## Grid & spacing
- List row height comfortable for scanning (not marketing card height).
- Detail view either takes a right panel or a dedicated route.

## Breakpoints
- Mobile: list-only view, detail on tap opens a new screen.
- Desktop: list + detail side by side.

## Accessibility
- Table sortable via clickable headers with `aria-sort` state.
- Row selection accessible via keyboard and clearly indicated.

## Anti-patterns
- Detail panel that opens over the list and hides context.
- Filters that require a save button.
