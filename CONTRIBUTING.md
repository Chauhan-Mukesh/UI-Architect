# Contributing

## Development principles
- Follow `plan.md` roadmap phases.
- Keep UX-first and accessibility-first decisions.
- Prefer semantic HTML, modern CSS, and progressive enhancement.

## Contribution flow
1. Pick a roadmap section or checklist item.
2. Add or improve knowledge modules in the relevant folder.
3. Keep changes scoped and update docs when needed.
4. Open a PR with a clear summary of additions.

## Content expectations
- Markdown modules must be practical and reusable.
- Avoid generic templates; describe context-aware decisions.
- Include accessibility and performance considerations in every module.
