# Travel — Common Components

## Core components
- Value-first hero with one dominant action
- Benefit/offer cards with fast comparison cues
- Process timeline or step-based guidance
- Trust modules (proof, guarantees, credentials)
- FAQ and objection handling
- Conversion form with progressive disclosure

## Component expectations
- Every component must support a measurable user decision.
- States (default/hover/focus/error/loading) must be accessible.
