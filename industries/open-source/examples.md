# Open Source — Example Direction

## Example page goals
- Land, explain, and get someone running the project in under two minutes.
- Signal legitimacy through community and maintenance evidence.

## Suggested example scenario
- Landing page: positioning + quick-start, features with code, why-it-exists, community, sponsors.
- Docs site: getting-started, guides, API, migration, changelog.

## Review checklist
- Can a developer copy an install command in the first fold?
- Is at least one real code example visible without scrolling?
- Are community signals current and honest?
