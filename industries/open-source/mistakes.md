# Open Source — Common Mistakes

- Marketing hero with no code visible.
- Install commands as images or non-copyable code.
- Vanity metrics (fake stars, inflated download counts).
- Docs hidden behind a login or newsletter wall.
- No dark mode when the audience lives in dark IDEs.
- Missing accessibility on syntax-highlighted code blocks.

## Specific risk
- Presenting the project as a product with hidden pricing when it is open source — erodes trust immediately.
