# Open Source — Common Components

## Core components
- Positioning hero: what / for whom / why (with immediate quick-start snippet)
- Install / quick-start block (copyable, multi-package-manager)
- Feature highlights with code examples
- Docs entry (getting-started, guides, API reference)
- GitHub metrics strip (stars, contributors, latest release)
- Sponsor / support module if applicable
- Community links (Discord, forum, mailing list)
- Contributor / roadmap block

## Component expectations
- Every code sample is real, syntax-highlighted, and copy-safe.
- Metric strips update dynamically or are removed if unmaintainable.
- Docs component works offline and prints reasonably.
