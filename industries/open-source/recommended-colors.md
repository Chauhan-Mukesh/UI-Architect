# Open Source — Recommended Colors

## Palette direction
- Developer-friendly neutrals with a single strong brand color.
- Support both light and dark themes at parity — dark is not an afterthought.
- Syntax-highlighting palettes should coexist with UI palette without visual conflict.

## Usage constraints
- Code-block contrast for both themes must pass AA.
- Do not rely on color alone for diff/error/success states — pair with icon.
- Terminal-style green-on-black used sparingly; not for body copy.
