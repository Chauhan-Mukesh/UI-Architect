# Open Source — Layout Preferences

## Recommended structure
1. Positioning line + install command + primary CTA
2. Why-it-exists / what-it-replaces
3. Feature highlights with code examples
4. Community and metrics
5. Docs entry and roadmap
6. Sponsors / footer with repo, license, contributors

## Layout rules
- Code blocks are content; give them layout weight.
- Docs navigation is persistent, searchable, keyboard-friendly.
- Mobile shrinks code samples but keeps copy-to-clipboard functional.
- Avoid marketing-first hero when the audience is engineering.
