# Artificial Intelligence — Layout Preferences

## Recommended structure
1. Positioning + live example
2. Capability grid tied to real tasks
3. Benchmarks or evaluation methodology
4. Pricing tied to usage
5. Safety / responsible-use content
6. Enterprise / security path
7. Docs and community

## Layout rules
- Interactive demo occupies a prominent, generous slot — not a modal buried in a link.
- Long-running requests must have skeleton or streaming feedback, not just a spinner.
- Safety content earns page real estate proportional to product risk.
- Mobile keeps demo functional or explicitly points to desktop for that section.
