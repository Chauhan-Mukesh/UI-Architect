# Artificial Intelligence — Common Components

## Core components
- Positioning hero: what task, what output, what audience
- Interactive demo or live example strip
- Feature or capability grid tied to concrete use cases
- Benchmarks / evaluation methodology (if claims made)
- Trust and safety module (data handling, model provenance, human review)
- Pricing tied to usage units (tokens, calls, seats)
- Docs entry for API/product
- Enterprise / security / compliance path when relevant

## Component expectations
- Any demo must be free of hidden rate-limiting that fails silently.
- Benchmarks are cited or explicitly labeled as internal.
- Safety content is not buried at the footer.
