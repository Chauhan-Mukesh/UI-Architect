# Developer Tools — Layout Preferences

## Recommended structure
1. Value proposition + primary action
2. Capability/offer overview
3. Proof and trust signals
4. Process and delivery expectations
5. FAQs and risk-reduction details
6. Final conversion block

## Layout rules
- Prioritize scan speed for high-intent users.
- Keep information hierarchy consistent between desktop and mobile.
- Avoid decorative complexity that hides decision-critical information.
